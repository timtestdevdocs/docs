## Request Parameters

### Body

<ParamField body="query" type="string" required>
  The search value. The maximum query character length is 400.
</ParamField>

<ParamField body="spell_check" type="boolean" default="true">
  Spell check the search query.
</ParamField>

<ParamField body="safe_search" type="string" default="moderate">
  Include offensive results in the search results
  <ul>
    <li>`moderate`</li>
    <li>`strict`</li>
    <li>`off`</li>
  </ul>
</ParamField>

<ParamField body="ai_overview" type="boolean" default="true">
  Include AI powered overview in the search results
</ParamField>

<ParamField body="byo_urls" type="string[]">
  Return results from the specified URLs.
</ParamField>

<ParamField body="country_code" type="string">
  Specify the country code for localized search results.
</ParamField>

<ParamField body="auto_scrape" type="boolean">
  Automatically scrape search results for more detailed information.
</ParamField>

<ParamField body="deep_research" type="boolean">
  Toggle deep research mode.
</ParamField>

<ResponseField name="deep_research_config" type="object">
  The deep research configuration.
  <Expandable title="Deep Research Config Object Structure" defaultOpen>
    <ResponseField name="max_depth" type="number" default="3">
      Specifies the depth of research by controlling the number of recursive iterations the framework performs. A higher value enables more thorough but longer searches.
    </ResponseField>

    <ResponseField name="max_breadth" type="number" default="3">
      Determines the breadth of research by limiting the number of parallel subqueries generated at each depth level. A higher value provides wider coverage of related topics.
    </ResponseField>

    <ResponseField name="max_output_tokens" type="number" default="32000">
      The maximum number of tokens in the generated output report.
    </ResponseField>

    <ResponseField name="target_output_tokens" type="number">
      The target number of tokens for the generated output report.
    </ResponseField>

  </Expandable>
</ResponseField>

## Usage Examples

### Basic Search

```javascript
import { JigsawStack } from "jigsawstack";

const jigsaw = JigsawStack({ apiKey: "your-api-key" });

const response = await jigsaw.web.search({
  query: "Artificial Intelligence applications",
  ai_overview: true,
  safe_search: "moderate"
});

console.log(response.ai_overview);
console.log(response.results);
```

### Advanced Search with Deep Research

```javascript
const response = await jigsaw.web.search({
  query: "Climate change impact on agriculture",
  deep_research: true,
  deep_research_config: {
    max_depth: 4,
    max_breadth: 5,
    max_output_tokens: 50000
  },
  country_code: "US",
  auto_scrape: true
});

console.log(response.ai_overview);
console.log(response.results);
```

### Search with Custom URLs

```javascript
const response = await jigsaw.web.search({
  query: "Latest advancements in renewable energy",
  byo_urls: [
    "https://www.energy.gov/eere/renewables",
    "https://www.irena.org/",
    "https://www.nrel.gov/"
  ],
  spell_check: true
});

console.log(response.results);
```

## Best Practices

1. Use `spell_check: true` to automatically correct misspelled queries.
2. Adjust `safe_search` based on your application's content requirements.
3. Utilize `deep_research` for comprehensive results on complex topics.
4. Use `byo_urls` to include specific trusted sources in your search results.
5. Set appropriate `country_code` for location-specific searches.
6. Enable `auto_scrape` for more detailed information from search results.

By leveraging these parameters, you can customize the AI Search functionality to best suit your application's needs and provide users with relevant, comprehensive, and safe search results.
